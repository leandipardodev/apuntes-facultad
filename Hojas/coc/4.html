CONCEPTOS DE SISTEMAS OPERATIVOS

La necesidad del "Software"

En síntesis, la distancia entre los usuarios (que tienen los problemas del mundo real) y las computadoras (que los podrían ayudar a resolver) requiere un puente lógico y conceptual que está constituido por el software.



La producción de sistemas de software (que constituyen el puente útil entre el usuario y la computadora) es una actividad industrial que requiere métodos, herramientas y procedimientos que se estudian a lo largo de la carrera.



Un modelo por capas desde la máquina al usuario

  1. Computadora (hardware)

  2. Sistema Operativo

    1. Sistema Operativo residente básico
    2. Funciones de propósito general para utilizar el hardware
    3. Funciiones de administración de recursos
  3. Utilitarios básicos
  4. Lenguajes de Programación de Aplicaciones
  5. Lenguajes Orientados a la Aplicación
  6. Sistemas de Software de propósito general
  7. Sistemas de Software dedicados
  8. Usuario (mundo real)

La primera capa (la electriónica de la computadora) depende totalmente del software para ser funcional, sin este no seria mas que un complejo sistema electronico sin ninguna funcion util

La segunda capa es el sistema operativo, nos permite comunicarnos con la computadora y utilizar eficientemente sus recursos. para analizarlo lo subdividimos en tres niveles: el s istema operativo residente básico, las funciones portables de control del hardware y las funciones de administración de tareas y recursos.
El primer nivel del sistema operativo (de ahora en adelante S.O) es la BIOS, es el mecanismo utilizado por la pc para darle las primeras instrucciones indispensables para poder comunicarnos con el, por ejemplo habilitar el teclado, verificar la pantalla. Estas funciones vienen incorporadas con el hardware y no se borran al apagar la maquina ya que estan almacenadas en la memoria ROM.
El segundo nivel del S.O se encarga de agregar funciones útiles para el usuario, por ejemplo poder tener un antivirus o que se muestren los archivos y elementos visuales entre otras muchas cosas.
El tercer niuvel del S.O se refiere esencialmente a las funciones de administración de recursos:
administrar la memoria principal, dispositivos de almacenamiento secundario, diferentes usuarios de la maquina, control ejecución aplicaciones,etc...

La tercera capa (Utilitarios básicos) se refiere a los programas sencillos; visualizacion de imágenes, bloc de notas, reproductor multimedia.

En la cuarta capa estan los lenguajes de programación de aplicaciones (Pascal, C, Java, C++). Un lenguaje de programación establece un modo de escribir instrucciones a una computadora en lenguaje "humano", este es traducido a lenguaje de máquina (binario).

En la quinta capa tenemos los lenguajes orientados a la aplicación. Estos son mas amigables con el usuario a la hora de escribir y no requieren de un nivel tan tecnico como lo pueden ser Pascal, C , etc... Un ejemplo de esto puede ser el autocad.


En la sexta capa tenemos los Sistemas de software de propósito general (tales como sistemas contables, de liquidación de sueldos, etc).
Desarrollar este tipo de sistemas (independientemente del lenguaje que se utilice) es la actividad más importante de la ingieria de Software.Su destino es lo que se llama el "mercado horizontal", es decir son de utilidad para una gran cantidad y clase de usuarios. por ejemplo un sistema de sueldos peude ser empleado por empresas de cualquier tipo en diferentes partes del mundo.

En la séptima capa tenemos los Sistemas de software dedicados. En este caso se trata de desarrollar un producto "a medida" para una organización, empresa o máquina. Por ejemplo, controladores de un robot o de un lavarropas; sistemas de cálculo financiero, sistemas de diagnostico de enfermedades. En estos casos el valor agregado del producto software es muy alto, y en general la solución es exclusiva o poco portable a otras organizaciones o productos.

Y la octava capa el usuario del mundo real sigue siendo un desafío muy complejo para la informática. Muchas veces los rpoblemas planteados no tienen (o no se encuentra) una solución eficiente utilizando computadoras y esto obliga a una permanente investigación y desarrollo de herramientas y productos de software.

Una definición de sistema operativo

En resumen un S.O es basicamente un sistema de control del hardware de una computadora, de administración de sus recursos físicos y de sus usuarios, asi como el control efectivo de la ejecución de los programas que en el se carguen.

  • El funcionamiento del S.O implica la existencia de al menos un programa qque está permanentemente ejecutándose junto con nuestras aplicaciones.
  • Esto significa que un subconjunto de los recursos de las computadoras son "tomados" por el S.O que se comunica directa o indirectamente con los programas de aplicación que se ejecutan para cooperar en la ejecución o retomar el control del hardware en caso de problemas.

¿qué hace un Sistema Operativo?

  • Comunicación con los periféricos
  • Control de autorización de usuarios
  • Control de la ejecución de programas
  • Control de concurrencia(gestión de procesos, Comunicación entre estos y sincronización)
  • Control de errores
  • Administración de memoria
  • Controles de seguridad de datos

La interfaz del usuario con el sistema operativo

La interfaz es el modo amigable para que el usuario interactue con el S.O, historicamente las interfaces estuivieron basadas en comandos formados por palabras claves predefinidas para ser interpretados por el sistema operativo.
estas interfaces se denominan orientadas a caracteres. Por ejemplo: CMD de windows, powershell o el antiguo S.O MSDOS
En los últimos años se han impuesto las interfaces gráficas que contienen imagenes representativas (por ejemplo de los archivos o dispositivos o de los comandos), llamadas iconos que se pueden organizar en menúes que se abren y cierran (menúes descendentes) y que pueden expandirse en presentaciones dentro de múltiples ventanas en la pantalla.
Las interfaces graficas tipo VIMA(o wimp en ingles) significa ventanas, iconos, menúes y apuntadores tienen diferentes ventajas entre las cuales estan:

  • Son intuitivas: siguen unos estandares a los que ya estamos acostumbrados por lo tanto no debemos aprender a usarlas
  • Son consistentes: Toda una gama de aplicaciones (por ejemplo office) tienen la misma forma de interfaz, lo que favorece el aprendizaje y la seguirdad del usauario en su utilización
  • Facilitan el autoaprendizaje al ser repetitivas.
  • Incorporan mecanismos de seguridad (permiten volver atras, no tomar acciones drásticas por error, etc.)
  • Incrementan la flexibilidad. en particular se puede usar simultánea o alternativamente el teclado o mouse

¿Cuál es el futuro?

  • La tendencia es a las interfaces naturales: poder hablar directamente con la computadora y tener una respuesta auditiva o grafica
  • inteligencia artificial aplicados al manejo de la vinculación entre el S.O y el usuario, interpretan las elecciones más frecuentes del usuario y los "deseos" segun el tipo de proceso a realizar. por ejemplo el asistente de presentaciones de Power point
